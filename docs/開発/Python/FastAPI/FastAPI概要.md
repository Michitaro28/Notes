# FastAPI概要
FastAPIはPythonで高速なWeb APIを構築するためのモダンなフレームワークで、日本語の情報は公式ドキュメント翻訳やコミュニティの貢献により充実しつつありますが、英語の情報へのアクセスも必要です。高いパフォーマンス、型ヒントを利用した自動データ検証・ドキュメント生成（Swagger UI/ReDoc）、非同期処理サポートが特徴で、DjangoやFlaskと比較してモダンで使いやすいと評価され、人気が急上昇しています。（AIより）

- [公式ドキュメント](https://fastapi.tiangolo.com/)

## 特徴・ポイント

- 型ヒント
    Pythonの型ヒントは、変数や関数の引数、戻り値がどのような型であるかを明示するための機能です。FastAPIでは、この型ヒントを利用して、APIが受け取るリクエストのデータ型や返すリスポンスのデータ構造を定義します。これにより、データ検証やエディタの補完が自動で行われ、開発効率が大幅に向上します。

- OpenAPI
    「OpenAPI」は、RESTful APIの仕様を記述するための基準フォーマットです。FastAPIは型ヒントを基に、自動的にこの「OpenAPI」を生成します。これにより、APIのエンドポイント（ネット上における特定のアクセスポイントのこと）、リクエスト。レスポンスの構造が明確に文書化され、APIの使用方法が容易に理解できるようになります。

- SwaggerUI
    「SwaggerUI」はOpenAPIの仕様書を基に、ブラウザ上で動作するインタクティブなAPIドキュメントを提供する。FastAPIでは、SwaggerUIが自動生成され、ドキュメントを見ながらAPIのテストをすることができます。

- スキーマ駆動開発
    APIの仕様・構造（スキーマ）を最初に定義し、それに基づいて開発を進める方法です。FastAPIでは、型ヒントとOpenAPIを利用して、SwaggerUIによるドキュメントを自動で生成されます。自動で生成されたドキュメントを「バックエンド」と「フロントエンド」の開発者が共有することで、開発作業効率を密接に連携しやすくなります。


## RESTful APIについて深掘り
    RESTful APIとは、Webサービス間でデータをやり取りするための設計原則（REST）に従ったAPIのことで、URLでリソースを識別し、HTTPメソッド（GET, POSTなど）を使って操作する、シンプルで拡張性の高い通信方式です。ステートレス（状態を持たない）であり、異なるアプリケーションが標準的なHTTPプロトコルで安全かつ効率的に連携できるため、Webやモバイルアプリ、IoTなどで広く使われています。

**RESTful APIの主な特徴**
    リソース指向: すべてのデータ（情報）を「リソース」とみなし、それぞれに固有のURL（URI）を割り当てます。
    HTTPメソッドの利用: リソースの操作（取得、作成、更新、削除）にGET, POST, PUT, DELETEなどのHTTPメソッドを適切に利用します。
    ステートレス: サーバーはクライアントの状態を保持せず、各リクエストは独立して処理されます（セッション管理が不要）。
    統一インターフェース: クライアントとサーバーは、標準的なHTTPプロトコルとデータ形式（JSON, XMLなど）で通信します。
    疎結合: クライアントとサーバーの実装が独立しており、互いに影響を与えずに進化できます。

**なぜ使われるのか？**
    シンプルで分かりやすい: URLとHTTPメソッドで直感的に操作できるため、開発者が理解しやすい。
    高い拡張性と柔軟性: 異なるプラットフォームや言語のアプリ間で簡単に連携できる。
    パフォーマンス: ステートレス設計やキャッシュ機能により、効率的な通信が可能。

**具体例**
    ECサイトで商品情報を取得する（GET /products/123）。
    SNSで新しい投稿を作成する（POST /posts）。
    RESTful APIは、Webシステムを「窓口」として外部に公開し、連携させるための現代的で標準的な手法と言えます。
